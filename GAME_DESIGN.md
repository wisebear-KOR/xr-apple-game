# XR Apple 10 — 기획서

> 갤럭시 XR(Android XR) / WebXR 기반 공간형 "사과 게임"
> 작성일: 2026-04-17

---

## 1. 게임 개요

### 1.1 한 줄 소개
내 방의 벽, 바닥, 가구 위에 숫자 사과가 돋아나고, 허공에서 손으로 사과들을 연결해 **합 10**을 만들어 터뜨리는 공간형 퍼즐 게임.

### 1.2 원작 레퍼런스
- 2D "사과 게임" (Fruit Box 류): 격자 안의 숫자 사과를 드래그 사각형으로 선택 → 선택한 사과들의 합이 10이면 제거 → 제한 시간 안에 최대 점수.

### 1.3 XR로 재해석한 핵심 변화
| 원작 (2D) | XR 버전 |
|---|---|
| 평면 격자 | **내 방 공간 자체가 보드** (벽/바닥/책상/소파 위에 사과 배치) |
| 마우스 드래그 사각형 | **손으로 허공에 3D 올가미(lasso) 그리기** 또는 양손으로 박스 영역 지정 |
| 고정 시점 | 내가 **걸어다니며** 사각지대의 사과를 찾아야 함 |
| 단일 난이도 | 공간 스캔 결과에 따라 **방마다 다른 레이아웃** |

## 2. 타겟 & 플랫폼

- **디바이스**: Samsung Galaxy XR (Android XR)
- **엔진**: WebXR + Three.js (GitHub Pages 배포)
- **필수 기능**: Passthrough AR, Plane Detection, Hand Tracking
- **선택 기능**: Eye Tracking, Depth Sensing / Occlusion, Spatial Anchors
- **세션 타입**: `immersive-ar` (패스스루 필수)

## 3. 핵심 게임 루프

```
[공간 스캔] → [사과 배치] → [탐색 & 선택] → [합=10 판정] → [점수 & 연쇄] → 반복 → [타임오버]
```

### 3.1 게임 진행 (1 라운드 = 90초)

1. **공간 스캔 (10초)**
   - WebXR `hit-test` / plane detection으로 주변 평면 인식
   - 감지된 평면을 푸른 윤곽선으로 시각화 → "여기에 사과가 자랍니다"
2. **사과 스폰**
   - 30~50개의 숫자 사과(1~9)를 감지된 평면 위/주변에 배치
   - 평면 종류별로 배치 전략을 다르게: 바닥엔 줄지어, 벽엔 격자, 책상 위엔 무더기
3. **선택 & 수확**
   - 플레이어는 움직이며 사과를 탐색
   - 한 손으로 허공에 **올가미(lasso)** 를 그리거나, 양손으로 **박스 영역**을 지정
   - 영역 안 사과들의 숫자 합 = 10 → 전부 터지며 점수 획득
   - 합이 10이 아니면 영역이 붉게 깜박이고 취소
4. **연쇄(Combo)**
   - 3초 이내에 다음 10을 만들면 콤보 배수 ×1.5, ×2.0 …
5. **타임오버 → 점수 집계**

### 3.2 점수 규칙
- 기본: 터뜨린 사과 수 × 10
- 콤보 배수 적용
- **공간 활용 보너스**: 한 번의 선택에서 서로 다른 평면(바닥+벽+책상)의 사과를 포함하면 +50%
- **원거리 보너스**: 내 위치에서 2m 이상 떨어진 사과를 포함하면 +20%

## 4. XR 기술 활용 포인트

단순히 "사과를 공중에 띄우는" 수준을 넘어, XR 고유 기능을 게임 메카닉으로 녹입니다.

### 4.1 Plane Detection — "내 방이 곧 게임판"
- 감지된 수평/수직 평면마다 사과 배치 패턴을 달리함
- 매 플레이마다 레이아웃이 바뀌므로 재미 유지

### 4.2 Hand Tracking — "손이 곧 컨트롤러"
- **검지 끝**으로 허공에 선을 그려 올가미 생성 (폐곡선이 되면 영역 확정)
- **엄지+검지 핀치**로 사과 단일 선택 토글
- **양손 핀치 → 벌리기**로 3D 박스 볼륨 확장

### 4.3 Depth / Occlusion — "실제 사물 뒤에 숨은 사과"
- 사과가 실제 의자 뒤로 가려져 보이지 않을 수 있음
- 플레이어가 고개를 움직여 시점을 바꿔야 찾을 수 있음 → 탐색의 재미

### 4.4 Eye Tracking (선택) — "시선 조준 모드"
- 보조 입력: 바라본 사과에 옅은 하이라이트
- 접근성 옵션: 손을 쓰기 어려운 사용자는 시선+핀치로 선택

### 4.5 Spatial Audio — "사과의 속삭임"
- 각 사과가 위치 기반 사운드를 방출 (숫자가 클수록 낮은 톤)
- 뒤에 있는 사과는 뒤에서 들림 → 소리로 숨은 사과 탐지
- 합이 10 성공 시 상쾌한 과즙 터짐 사운드 (방향성 있음)

### 4.6 Spatial Anchors — "이어하기"
- 세션 종료 시 사과 위치를 앵커에 저장
- 다음에 같은 공간에서 앱을 켜면 어제 남긴 사과가 그대로 있음

### 4.7 Passthrough 조명 — "내 방 분위기 반영"
- 실내 밝기에 따라 사과의 광택을 조절 (낮엔 빨갛게, 밤엔 은은하게 발광)

## 5. 모드 제안

| 모드 | 설명 | 시간 |
|---|---|---|
| **클래식** | 90초 동안 최대 점수 | 90s |
| **젠(Zen)** | 시간 제한 없음, 사과가 계속 리필, 명상하듯 플레이 | ∞ |
| **헌팅** | 사과가 방 곳곳에 숨겨져 있음, 먼저 찾아야 함 | 3분 |
| **멀티(향후)** | 같은 방 공간에서 친구와 경쟁 (WebRTC) | 2분 |

## 6. UI / UX

### 6.1 HUD (최소화 원칙)
- 좌측 손목 위: 점수 / 남은 시간 (손목을 위로 돌릴 때만 표시)
- 시야 중앙엔 HUD 없음 → 몰입 유지

### 6.2 온보딩
1. "방을 천천히 둘러봐 주세요" (스캔 유도)
2. "손가락으로 허공에 원을 그려보세요" (올가미 튜토리얼)
3. "2와 8, 합쳐서 10!" (첫 성공 연출)

### 6.3 피드백
- 선택 중: 올가미 라인이 빛나며 현재 합계가 라인 중앙에 실시간 표시
- 합=10: 사과들이 안쪽으로 수축 → 과즙 파티클 → 숫자 "+120" 팝업
- 합≠10: 라인이 붉게 깜박 후 소멸

## 7. 기술 스택 & 구조

```
xr-test/
├─ index.html              # 엔트리, WebXR 세션 시작
├─ src/
│  ├─ main.js              # Three.js 부트스트랩
│  ├─ xr/
│  │  ├─ session.js        # XR 세션 관리
│  │  ├─ planes.js         # plane detection
│  │  ├─ hands.js          # hand tracking + 제스처
│  │  └─ anchors.js        # spatial anchors
│  ├─ game/
│  │  ├─ apple.js          # 사과 엔티티
│  │  ├─ spawner.js        # 평면 기반 사과 배치
│  │  ├─ lasso.js          # 올가미/박스 선택
│  │  ├─ scoring.js        # 점수/콤보
│  │  └─ audio.js          # 공간 오디오
│  └─ ui/
│     └─ hud.js
├─ assets/
│  ├─ models/ (glTF, Draco 압축)
│  ├─ sounds/
│  └─ textures/
└─ .github/workflows/pages.yml  # GitHub Pages 자동 배포
```

**핵심 라이브러리**
- `three` — 렌더링
- `three/examples/jsm/webxr/*` — WebXR 빌트인 모듈 (컨트롤러, 핸드)
- (선택) `cannon-es` — 사과 물리 (살짝 흔들리는 연출)

## 10. 확장 아이디어 (백로그)

- **계절 테마**: 가을엔 낙엽 파티클, 겨울엔 눈 내림
- **파워 사과**: 특정 사과(★)를 포함하면 점수 2배
- **도전 과제**: "한 번에 5개 이상 연결", "벽 사과만으로 10 만들기"
- **공간 기록**: 같은 방에서의 하이스코어를 앵커와 함께 저장, 리더보드
- **협동 모드**: 로컬 멀티(같은 방 2인 공유 공간)
